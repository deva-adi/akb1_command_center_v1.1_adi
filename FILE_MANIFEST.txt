TECHNICAL SPECIFICATIONS:
=========================

DOCKER BUILD PROCESS:
Stage 1 (Frontend):
  Base image: node:20-alpine
  Working directory: /app/frontend
  Steps:
    1. Copy frontend/ directory
    2. npm install
    3. npm run build
  Output: /app/frontend/dist

Stage 2 (Production):
  Base image: python:3.11-slim
  Working directory: /app
  Dependencies installed: sqlite3, curl
  Steps:
    1. Copy backend/requirements.txt
    2. pip install Python dependencies
    3. Copy backend code
    4. Copy frontend dist from Stage 1 → frontend_dist/
    5. Create /data directory
    6. Expose port 8000
    7. Setup health check
  Start command: uvicorn backend.main:app --host 0.0.0.0 --port 8000

CI/CD PIPELINE:
Trigger: Push to main branch
Platform: GitHub Actions
Cloud deployment: Railway
Deployment mechanism: railway up CLI command
Authentication: RAILWAY_TOKEN secret

SERVICE ARCHITECTURE:
Port 8000:
  /api/* → FastAPI backend routes
  /assets/* → Static React build files
  /* → SPA fallback (index.html)

Database:
  Type: SQLite
  Location: /data/akb1_command_center.db
  Persistence: Docker volume ./data:/data
  Auto-initialization: On first startup
  Auto-seeding: If database is empty

HEALTH CHECK:
Endpoint: GET /api/health
Docker method: curl http://localhost:8000/api/health
Interval: 30 seconds
Timeout: 10 seconds
Start period: 5 seconds
Retries: 3 before marking unhealthy

DEPLOYMENT INSTRUCTIONS:
========================
Local Testing:
  $ cd /akb1-command-center-v3
  $ docker-compose up --build
  $ # Access at http://localhost:8000

Railway Deployment:
  1. Create Railway account (https://railway.app)
  2. Create Railway project
  3. Generate Railway token
  4. Add RAILWAY_TOKEN to GitHub Secrets
  5. Push to main branch: git push origin main
  6. GitHub Actions will automatically deploy
